## Язык программирования

### Общая характеристика и семантика

В проекте реализован императивный язык с C-подобным синтаксисом, ориентированный на векторную обработку данных.

**Основные концепции:**
- **Типизация**: Статическая. Поддерживаются 32-битные знаковые целые числа (`int`) и статические одномерные массивы (`int[]`).
- **Строки и массивы**: В языке нет отдельного типа `string`. Все строковые литералы и массивы материализуются в памяти как **`int[]`**. Строки хранятся в формате **pstr** (первое слово — длина, далее — коды символов).
- **Области видимости**: Лексические. Переменные `let` видны только внутри своего блока `{}`.
- **Управляющие конструкции**: 
  - `if (cond) { ... } else { ... }`
  - `while (cond) { ... }`
  - **Short-circuit**: Операторы `&&` и `||` реализуются транслятором через условные переходы с short-circuit семантикой.
- **Функции**: Программа состоит из набора функций. 

### Текущая реализация и ограничения

В текущей версии реализации (`src/hl_logic.py`) действуют следующие ограничения:
1. **Точка входа**: Реализована генерация кода только для функции `function main()`.
2. **Пользовательские функции**: Декларации поддерживаются грамматикой, но вызовы и передача параметров в кодогенераторе не реализованы.
3. **Типы данных**: Символы `'A'` и логические значения `true`/`false` являются псевдонимами для `int`.
4. **Проверки**: Runtime-проверка границ массива и переполнения памяти отсутствует.
5. **Арифметика**: Встроенная функция `print_num` поддерживает только положительные целые числа. Деление на ноль вызывает Python `ZeroDivisionError`, останавливающее симулятор.

### Встроенные функции
- `print(int)` — вывод ASCII-символа.
- `print_num(int)` — десятичный вывод положительного целого.
- `print_str(int[])` — вывод строки pstr.
- `read()`, `readln()` — чтение из `IN_PORT`.
- `len(arr)` — заменяется целочисленным литералом во время трансляции.
- `vload(arr, idx)`, `vadd(a, b)`, `vstore(arr, idx, vec)` — операции векторного расширения.

## Организация памяти

### Модель адресации
Процессор использует **гарвардскую архитектуру** с раздельной памятью инструкций и данных.

- **Единица адресации**: 32-битное машинное слово. Все адреса и смещения в ISA указаны в словах.
- **Память инструкций**: Начинается с `0`. Код **не является перемещаемым** (non-relocatable) из-за использования абсолютных 26-битных переходов (`J`).
- **Память данных**: Адресуется через GPR (база `gp`).
  - `0x0000`: Зарезервировано.
  - `0x0001 - 0x1FFF`: Статические данные (строки, массивы, переменные). Регистр `gp` инициализируется значением `1`.
  - `0x2000` (8192): База стека. ISA поддерживает инструкции стека, но HL-компилятор их не использует.

### Memory-mapped I/O (MMIO)
Устройства ввода-вывода отображаются в 32-битное адресное пространство данных:

| Имя | Логический адрес (HEX) | Кодирование в инструкции (sign-ext 11-bit) | Доступ |
| :--- | :--- | :--- | :--- |
| `IN_PORT` | `0xFFFFFFF0` | `-16` (`0x7F0`) | Чтение |
| `OUT_PORT` | `0xFFFFFFF4` | `-12` (`0x7F4`) | Запись |

**Механика**: В инструкциях `LW`/`SW` адрес вычисляется как `rs1 + sign_ext(imm11)`. Для доступа к портам компилятор использует смещения `-16` и `-12` относительно `zero` (x0).

## Система команд

### Особенности ISA
- **RISC**: Модель load/store. Арифметика только над регистрами.
- **Регистры**: 32 скалярных GPR (32 бита) и 8 векторных регистров (4 lane × 32 бита).
- **Кодирование констант**: Используется пара `LUI rd, imm21` (сдвиг на **11 бит**) + `ADDI rd, rd, imm11`. Сдвиг на 11 позволяет 21-битной и 11-битной частям полностью перекрыть 32-битное слово.

### Выполнение (Ticks)
Машина является **microcoded (µROM)**. Каждый такт `tick()` — это одна микроинструкция.
- Скалярные R/I: 2 такта (Fetch + Execute).
- Память (LW/SW): 3-4 такта.
- Векторные (VLD/VST): 6 тактов.

## Векторное расширение

### Уровни реализации
1. **Язык**: Операции над массивами (`+`, `-`) и встроенные функции `vload`/`vstore`.
2. **Компилятор**: Реализует **Strip-mining**. Массивы любой длины $N$ разбиваются на блоки по 4 (векторные инструкции) и скалярный "хвост" для остатка.
3. **Hardware**: **Vector ALU** обрабатывает 4 lane за 1 такт. Доступ к памяти последовательный (4 такта на 4 слова) из-за однопортовой памяти данных.
