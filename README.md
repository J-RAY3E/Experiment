## Architecture
The system is built as a **Harvard architecture** model with a **microcoded Control Unit**. It simulates a RISC processor at the tick level, ensuring complete cycle-accurate execution of instructions.

- **DataPath**: Handles register files, the ALU, and Data Memory.
- **ControlPath**: Implements the microcode ROM that interprets instructions and generates control signals.
- **I/O**: Uses Memory-Mapped I/O (MMIO) for streaming data via `IN_PORT` (`0xFFFFFFF0`) and `OUT_PORT` (`0xFFFFFFF4`).

## Memory
The model uses word-addressed memory for both instructions and data:
- **Instruction Memory**: Stores fixed-length 32-bit instructions.
- **Data Memory**: Includes static data and a stack that grows downwards from `0x1000`.

## ISA
The RISC-IV ISA is a 32-bit architecture supporting 32 scalar registers and 8 vector registers.
- **Scalar**: Arithmetic, logical, branch, jump, and memory load/store operations.
- **System**: Supports `HALT` to terminate simulation.

## Vector Extension
This experiment includes a custom vector unit:
- **Registers**: 8 registers (`v0`-`v7`), each containing 4 × 32-bit lanes.
- **Operations**: Includes `VADD`, `VSUB`, `VMUL`, `VDIV`, and `VCMP`.
- **Data Movement**: Supports `VLD` and `VST` for bulk memory access.
- **Performance**: Designed to process arrays by lanes, significantly reducing the tick count compared to scalar loops.

## Toolchain
- **Translator**: Converts high-level source or assembly into binary images.
- **Simulator**: `src/Machine.py` simulates the datapath based on the control signals generated by `ControlPath.py`.
- **Testing**: Golden Tests are used for automated validation.
  - Run all tests: `python tests/test_golden.py`
  - Update tests: `python tests/test_golden.py --update`
- **Verification**: Each test compares the actual output, machine state journal, and listing against a validated YAML reference.
